# Bazaar Setu Free Staging Deploy

This is the current no/near-zero-cost path for Bazaar Setu while the product is still being built. It is good for demos, QA, and investor/internal review. It is not the final production architecture.

Do not use Fynd, GoFynd, or any shared company account/resource for this setup.

## Free Stack

- API: Render free web service from `render.yaml`.
- Admin/support web: Render free web service from `render.yaml`.
- Database: Neon Free Postgres.
- Redis: Upstash Free Redis over REST.
- OTP: `OTP_DELIVERY_MODE=mock`.
- Address location: `MAPS_PROVIDER=browser`, using device/browser GPS plus manual address entry.
- Payments: `PAYMENTS_PROVIDER=mock`.

Expected limitations:

- Render free services can sleep after inactivity, so first request may be slow.
- Neon free compute can scale to zero, so first DB request may be slow.
- Upstash free Redis has monthly command/storage limits.
- Mock OTP returns a demo OTP for staging login and must not be used for real launch.
- Browser/manual address mode is enough for staging; Google Maps is required before real operations.

## 1. Create Neon Free Postgres

1. Open `https://console.neon.tech/`.
2. Create a free project named `bazaar-setu-free-staging`.
3. Create/use a database named `bazaarsetu_staging`.
4. Copy the direct Postgres connection string.
5. Ensure the URL includes SSL, for example `?sslmode=require`.

Use this value as Render API `DATABASE_URL`.

## 2. Create Upstash Free Redis

1. Open `https://console.upstash.com/`.
2. Create a free Redis database named `bazaar-setu-free-staging`.
3. Open the database Details page.
4. Copy `UPSTASH_REDIS_REST_URL` and `UPSTASH_REDIS_REST_TOKEN` from the REST connection section.

Use these values as Render API `UPSTASH_REDIS_REST_URL` and `UPSTASH_REDIS_REST_TOKEN`.

## 3. Generate Staging Secrets

Run locally:

```bash
cd /Users/rohitvirendrayadav/bazaarsetu-mvp/bazaar-setu-app
npm run secrets:free-staging
```

Paste the generated values into the Render API service environment variables:

- `JWT_SECRET`
- `OTP_CODE_PEPPER`
- `ADMIN_BOOTSTRAP_TOKEN`

Keep `ADMIN_BOOTSTRAP_TOKEN` private. Rotate/remove it after the first Admin user is created.

## 4. Create Render Services From Blueprint

1. Open `https://dashboard.render.com/`.
2. Create a new Blueprint.
3. Connect GitHub repo `rohity008-RY/MVP-`.
4. Use branch `main`.
5. Render should detect `render.yaml` at repo root.
6. Create both services:
   - `bazaar-setu-api-free`
   - `bazaar-setu-admin-free`

The blueprint uses `plan: free`, Docker builds, Singapore region, and the API container runs Prisma migrations before starting the server.

## 5. Render API Environment Variables

Most values are defined in `render.yaml`. Fill these manually in Render:

- `DATABASE_URL`: Neon direct Postgres URL.
- `UPSTASH_REDIS_REST_URL`: Upstash REST URL, e.g. `https://...upstash.io`.
- `UPSTASH_REDIS_REST_TOKEN`: Upstash standard REST token.
- `JWT_SECRET`: generated by `npm run secrets:free-staging`.
- `OTP_CODE_PEPPER`: generated by `npm run secrets:free-staging`.
- `ADMIN_BOOTSTRAP_TOKEN`: generated by `npm run secrets:free-staging`.
- `API_BASE_URL`: final API URL, for example `https://bazaar-setu-api-free.onrender.com`.
- `CORS_ORIGINS`: comma-separated app origins, start with `https://bazaar-setu-admin-free.onrender.com`.

These are intentionally free/staging defaults:

- `OTP_DELIVERY_MODE=mock`
- `MAPS_PROVIDER=browser`
- `PAYMENTS_PROVIDER=mock`
- `DEMO_AUTH_ENABLED=false`

## 6. Render Admin Environment Variables

Set:

- `NEXT_PUBLIC_API_BASE_URL`: final API URL, for example `https://bazaar-setu-api-free.onrender.com`.

If the first Admin deploy happens before the API URL is known, update this variable after the API service is created and redeploy Admin.

## 7. Smoke Test

After deploy:

```bash
curl https://bazaar-setu-api-free.onrender.com/api/health
curl https://bazaar-setu-api-free.onrender.com/api/ready
curl https://bazaar-setu-admin-free.onrender.com/health
```

`/api/ready` should show:

- `redisConfigured: true`
- `redisMode: "upstash-rest"`
- `otpDeliveryMode: "mock"`
- `mapsProvider: "browser"`
- `paymentsProvider: "mock"`
- `adminBootstrapConfigured: true`

## 8. Bootstrap First Admin

Run once:

```bash
curl -X POST "https://bazaar-setu-api-free.onrender.com/api/auth/admin/bootstrap" \
  -H "content-type: application/json" \
  -H "x-admin-bootstrap-token: $ADMIN_BOOTSTRAP_TOKEN" \
  -d '{"phone":"+91XXXXXXXXXX","name":"Ops Admin","email":"ops@example.com"}'
```

Then rotate/remove `ADMIN_BOOTSTRAP_TOKEN`.

## 9. Later Paid Upgrade Path

When Bazaar Setu is ready for real operations:

- Move API/Admin to Cloud Run or paid Render.
- Move address search to Google Maps/Places with restricted keys.
- Move OTP to a real SMS/WhatsApp provider.
- Move payments to Razorpay live mode with refunds/webhooks.
- Add object storage for documents/product photos.
- Add monitoring, backups, PITR, alerts, and audit logs.
